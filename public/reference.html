References

This section provides details about the third-party APIs, libraries, and code utilized in our project.

APIs

Axios

Axios is a promise-based HTTP client for the browser and Node.js. We use Axios to handle HTTP requests and responses in our project, making it easier to communicate with our backend server.

JWT (JSON Web Token)

JWT is used for securely transmitting information between parties as a JSON object. We use JWT for authentication and authorization in our application.

Libraries

bcryptjs

bcryptjs is a library to help you hash passwords. We use bcryptjs to securely hash and store user passwords in our database.

Express

Express is a minimal and flexible Node.js web application framework that provides a robust set of features for web and mobile applications. We use Express to build our server-side application.

Mongoose

Mongoose is an Object Data Modeling (ODM) library for MongoDB and Node.js. We use Mongoose to interact with our MongoDB database, providing a schema-based solution to model our application data.

Morgan

Morgan is an HTTP request logger middleware for Node.js. We use Morgan to log HTTP requests and errors, which helps in monitoring and debugging our application.

dotenv

dotenv is a zero-dependency module that loads environment variables from a .env file into process.env. We use dotenv to manage environment variables in our project.

body-parser

body-parser is a Node.js body parsing middleware. We use body-parser to parse incoming request bodies in a middleware before your handlers, available under the req.body property.

cookie-parser

cookie-parser is a middleware which parses cookies attached to the client request object. We use cookie-parser to parse cookie headers and populate req.cookies with an object keyed by the cookie names.

serve-favicon

serve-favicon is a favicon serving middleware with caching. We use serve-favicon to serve the favicon of our application.

Code

Bootstrap Templates

We utilized components from two main Bootstrap templates to enrich our user interface. These templates facilitated a quicker development process, allowing us to concentrate on functionality and providing us with pre-designed elements that enhanced the visual appeal and intuitiveness of our application interface. We extend our gratitude to the creators of these templates.

  • NiceAdmin - This template provided us with foundational UI components. More details and documentation can be found at NiceAdmin. Components incorporated from this template include:
    • Buttons
    • Summary cards
    • Charts
    • Navigation bar
    • Table
  • SoftLand - A template that contributed to the landing page design elements. Visit SoftLand for more information. Components utilized from this template include:
    • Landing page animations
    • Buttons
    • Dropdowns
